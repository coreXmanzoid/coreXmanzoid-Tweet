Introduction

Overview

ChatFlick ("we", "us", or "our") is a social microblogging platform built by coreXmanzoid. This Privacy Policy explains what personal information we collect, why we collect it, and how you can exercise control over it.

By creating an account or using any part of ChatFlick, you agree to the practices described in this document. If you disagree with any part of this policy, please do not use the platform.

ChatFlick does not sell your personal data to advertisers or third-party marketers. Our platform is entirely ad-free within our own products.

Usage

How We Use Your Data

Your data is used exclusively to operate, improve, and secure ChatFlick. Specific uses include:

🔐 Account authentication 📰 Personalising your feed 🔔 Sending in-app & push notifications 🤖 Powering Manzoid-AI context 👥 Building social graph suggestions 🛡️ Admin moderation & safety 📧 Email verification & password reset 📦 Account data export 🎨 Applying your theme preference 📊 Follower milestone alerts

We do not use your data to build advertising profiles, train third-party machine learning models, or share it with data brokers.

Sign-in

Authentication & Account Security

Passwords are hashed using PBKDF2-SHA256 before storage. We never store or transmit plaintext passwords.

We support two sign-in methods:

Method Data handling
Email + Password Credentials verified locally. A signed time-limited token (20-minute expiry) is used for email verification and password reset flows. Reset requests return the same response whether an email exists or not, to prevent user enumeration.
Google OAuth We receive your name and email from Google's OpenID Connect endpoint. No Google access tokens are stored after authentication. Accounts created via Google are automatically marked as Verified.

All authentication flows require Cloudflare Turnstile CAPTCHA validation to prevent automated abuse.

Content

Posts & User Content

Content you post — including text, hashtags, and @mentions — is stored in our database and displayed to other users according to your privacy settings. Hashtags are extracted and stored separately to enable discovery features.

Posts can be edited or deleted only within a 3-minute window after creation. Deleting a post cascades — all associated comments and likes are removed from our database.

@Mentions create structured references stored as JSON alongside the post. Mentioned users may receive in-app notifications. Engagement data (likes, reposts, shares, comment counts) is stored as running counters and per-user tracking arrays to ensure uniqueness.

Alerts

Notifications & Push Tokens

If you grant notification permission, your browser provides a Firebase Cloud Messaging (FCM) token. This token is stored securely in your user record and used solely to deliver push notifications about activity on ChatFlick.

You can disable push notifications at any time in Settings → Notifications. Revoking browser notification permission will prevent push delivery. You control which event types trigger notifications (likes, comments, follows, reposts, mentions).

Self-notifications are suppressed — you will not receive a notification for your own actions. Push delivery silently fails if Firebase is not configured or if your device token has expired; we do not retry or log failed attempts against your profile.

AI Features

Manzoid-AI Assistant

ChatFlick includes an AI chat assistant powered by the Groq API (Llama 3.1-8b-instant model). When you send a message to Manzoid-AI:

What happens Detail
Message limit Authenticated chat is limited to 130 characters per message. The public endpoint is limited to 100 characters.
Short-term memory The last 2 exchanges are held in server process memory to provide context. This data is never written to our database.
Session resets Chat history is lost when the server restarts. There is no persistent conversation storage.
Third-party processing Your messages are sent to Groq's API for inference. Groq's own privacy policy applies to their processing.
Availability The AI feature depends on external API connectivity. Unavailability is surfaced as an error and does not affect your account data.

Security

Data Security

We implement the following protective measures:

Measure Details
Data Management SSL/TLS encryption, hashed passwords, restricted access, role-based permissions, automated backups.
Password hashing PBKDF2-SHA256 with a random salt of length 8 applied to every password before storage.
Signed tokens Email verification and password-reset links use itsdangerous URLSafeTimedSerializer with a 20-minute expiry.
Session management Authenticated sessions are managed server-side via Flask-Login. Session cookies are bound to the application's secret key.
Authorization checks Sensitive operations (profile upload, notification marking, post modification) verify that the requesting user matches the resource owner before proceeding.
CAPTCHA Cloudflare Turnstile is required on sign-up, login, and password-reset request flows.
Cloudinary images Profile images are served over HTTPS and transformed/resized server-side before storage.

No system is perfectly secure. In the event of a data breach affecting your personal information we will notify affected users as required by applicable law.
